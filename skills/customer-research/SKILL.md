# customer-research

客户背景调研与自动搜索，采用**渐进式披露 + 引导式互动**策略。

**核心理念：**
1. 不问业务场景，直接调研
2. 基础报告后给 2-3 个深度方案
3. 方案有"剧透"（可能发现什么）
4. 花钱 API 留在深度调研用

---

## 渐进式流程

```
用户：调研一下 XX 公司
    ↓
Phase 1: 基础调研（自动执行，不问）
  - 网络搜索（免费）
  - 工商基本信息查询（¥0.10）
    ↓
Phase 2: 基础报告 + 2-3 个深度方案
  - 方案 1：顺藤摸瓜查股东（免费搜索）
  - 方案 2：企业深度体检（¥3.00，花钱 API）
  - 方案 3：风险记录排查（免费）
    ↓
Phase 3: 用户选择 → 执行对应方案
  - 选免费 → 用网络搜索
  - 选花钱 → 确认后再执行
    ↓
Phase 4: 综合输出
```

## 企业类型分类（决定调研重点）

| 类型 | 特点 | 调研重点 |
|------|------|---------|
| **资源型** | 靠资源优势，坐吃山空 | 储量/品位/开采年限/采矿权 |
| **生产型** | 代工/生产制造 | 产能/产量/工艺/客户 |
| **贸易型** | 赚价差，流通环节 | 销售额/渠道/代理品牌/毛利 |
| **平台型** | 赚账期/杠杆/流量 | GMV/用户数/账期/资金池 |
| **科技型** | 技术驱动，研发投入 | 专利/研发团队/客户案例 |

## 业务场景（AI 自动判断关切点）

| 业务 | 核心问题 | 调研重点 |
|------|---------|---------|
| **商品酒销售** | 消费能力/采购习惯 | 员工规模/福利预算/接待用量 |
| **会员招募** | 行业地位/支付能力 | 行业排名/纳税等级/融资情况 |
| **定制酒** | 消费场景/决策链 | 年会/礼品/接待/决策部门/预算 |
| **基酒投资** | 投资偏好/现金流 | 投资历史/资产配置/现金流/决策人 |
| **老酒交易** | 收藏意识/资金实力 | 收藏历史/拍卖参与/资金实力 |

**注意：** 用户不指定业务时，AI 自动覆盖所有业务场景的关切点。

## Phase 1: 基础调研（自动执行）

**数据源：**
- 网络搜索（免费）
- 官网/新闻详情页（免费）
- 工商基本信息查询（¥0.10）

**获取信息：**
- 企业全称/法人/成立日期
- 注册资本/经营状态
- 主营业务/官网
- 上市情况/纳税等级（网络搜索）
- 员工规模（网络搜索/工商）
- 股东信息（仅名称和持股比例，免费）
- 对外投资（仅企业名称，免费）

**输出：** 基础报告 + 2-3 个深度调研方案

## 💡 深度线索识别原则

**完成基础调研后，AI 应自主思考：**

> "哪些信息可能影响客户的采购决策能力？哪些信息可能带来额外业务机会？"

**常见深度线索类型（仅供参考，非穷举）：**
- 股东/投资方背景（可能有关联资源或业务协同）
- 对外投资/分支机构（可能有多主体需求）
- 资质/荣誉（可能反映企业实力和采购偏好）
- 风险信息（可能影响支付能力或合作优先级）

**AI 应输出：** 1-3 个最值得深挖的线索 + 简要理由

## 错误处理

### 数据源失败
- 网络搜索失败 → 重试 1 次 → 提示用户
- 工商查询失败 → 提示余额不足 → 降级到免费方案

### 信息不足
- 基础信息不足 → 如实报告，建议深度调研
- 深度信息不足 → 建议人工补充

### 用户取消深度调研
- 尊重用户选择
- 输出基础报告
- 建议后续人工补充

## 🔗 与 customer-mgr 联动

**调研完成后自动执行：**

```python
# 1. 检查客户是否存在
customer = customer_mgr.find_customer(company_name)

if customer:
    # 2. 客户存在 → 关联调研报告 + 更新画像
    customer_mgr.link_research(company_name, research_data)
    print(f"✅ 调研报告已关联到客户档案：{company_name}")
else:
    # 3. 客户不存在 → 询问是否创建

    print("是否创建客户档案？")
    print("  - 回复'创建'或'是' → 调用 customer-mgr 创建")
    print("  - 回复'稍后'或'否' → 仅保存调研报告")
```

- 基础调研完成后
- 深度调研完成后
- 综合报告输出前

1. 调研报告 → `03-调研报告/{date}-{type}.md`
