# 搜索策略详解

## 搜索词构建原则

### 1. 分层搜索策略

**第一层：基础信息（必做）**
```
"{公司名} 上市 纳税 规模 员工"
```
目标：快速判断企业实力

**第二层：业务信息（必做）**
```
"{公司名} 官网 主营业务 产品"
```
目标：了解主营业务

**第三层：行业地位（选做）**
```
"{公司名} 行业地位 排名 龙头"
```
目标：判断行业影响力

**第四层：新闻舆情（选做）**
```
"{公司名} 最新 新闻 2025 2026"
```
目标：了解最新动态

## 搜索词优化技巧

### 精确匹配
```
"贵州茅台酒厂（集团）有限责任公司"  // 全称精确匹配
```

### 排除干扰
```
华兴集团 -股票 -代码  // 排除股票相关信息
```

### 限定范围
```
多彩贵州城 餐饮 site:gz.gov.cn  // 限定政府网站
```

### 时间限定
```
茅台 2025 新闻  // 限定最新信息
```

## 信息提取优先级

### 高优先级（必须提取）
1. **上市情况** — A 股/港股/美股/未上市
2. **纳税等级** — 省级/市级/县级纳税大户
3. **员工规模** — 具体数字或范围

### 中优先级（尽量提取）
4. **注册资本** — 工商注册信息
5. **年营收** — 财报或新闻披露
6. **主营业务** — 官网或百科描述

### 低优先级（可选提取）
7. **行业排名** — 行业协会/媒体评选
8. **发展历程** — 成立时间、里程碑
9. **企业荣誉** — 奖项、认证

---

## 搜索结果判断

### 信息充足
- ✅ 至少 2 个高优先级信息
- ✅ 至少 1 个中优先级信息
- ✅ 有可靠信息来源（官网、百科、新闻）

**处理：** 生成完整报告

---

### 信息不足
- ⚠️ 只有 1 个高优先级信息
- ⚠️ 信息来源不可靠

**处理：** 生成简化报告，标注信息不足

---

### 无结果
- ❌ 无任何有效信息
- ❌ 搜索结果为空

**处理：** 提示用户确认企业名称

---

## 多结果处理

### 场景：多个同名公司

**示例：** 搜索"华兴集团"返回：
1. 华兴控股集团（北京，投资集团）
2. 河南华兴集团（郑州，房地产）
3. 福建华兴集团（福州，茶业）

**处理策略：**
1. 检测搜索结果中的地域关键词
2. 检测搜索结果中的行业关键词
3. 向用户展示多个选项，请求确认

**话术：**
```
搜索到有几家叫"华兴集团"的公司：

1. 华兴控股集团 - 北京 - 产业投资
2. 河南华兴集团 - 郑州 - 房地产
3. 福建华兴集团 - 福州 - 茶业

请问是哪一家？
```

## API 使用优化

### 减少调用次数
- 合并搜索词（如"上市 纳税 规模"合并为一个查询）
- 优先使用高价值搜索词
- 信息充足时提前终止搜索

### 配额管理
- 每次调研最多消耗 5 个 credit（5 轮搜索）
- 简单企业 2-3 轮即可
- 记录每次消耗的 credit 数量

---

## 错误处理

### 搜索失败
- 网络超时 → 重试 1 次
- API 配额不足 → 提示用户
- 无搜索结果 → 尝试简化搜索词

### 信息提取失败
- 正则匹配失败 → 使用关键词匹配
- 多个冲突信息 → 优先最新/最权威来源
