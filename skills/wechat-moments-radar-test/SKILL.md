# 微信分析雷达

执行微信数据分析。默认场景是每日朋友圈关系维护提醒，目标是帮助用户快速判断“今天谁值得看、为什么、轻轻做什么”。

---

## 数据源

数据根目录：`~/.openclaw/`

```text
~/.openclaw/
├── raw/
│   ├── moments-analysis/                    ← L0 朋友圈原文
│   │   └── YYYYMMDD.json
│   ├── extracted_moment/                    ← L1 朋友圈抽取结果
│   │   ├── moment_extracted_YYYYMMDDHH.jsonl
│   │   └── checkpoint.json
│   ├── chat-analysis/                       ← L0 私聊原文
│   │   └── chat_YYYYMMDDHH.jsonl
│   ├── extracted_chat/                      ← L1 私聊抽取结果
│   │   ├── chat_extracted_YYYYMMDDHH.jsonl
│   │   └── checkpoint.json
│   ├── group-chat-analysis/                 ← L0 群聊原文
│   │   └── group_chat_YYYYMMDDHH.jsonl
│   └── extracted_group_chat/                ← L1 群聊抽取结果
│       ├── group_chat_extracted_YYYYMMDDHH.jsonl
│       └── checkpoint.json
├── customers/                               ← 客户历史
│   └── {wxid}/
│       └── wechat-analysis/
│           ├── {wxid}_moment_extracted.jsonl
│           ├── {wxid}_chat_extracted.jsonl
│           └── {wxid}_relations.jsonl
└── reports/                                 ← 输出报告
    ├── radar_YYYYMMDD.md
    ├── report_YYYYMMDDHH.md
    └── actions_YYYYMMDDHH.json
```

读取以下数据，并按职责使用：

- 朋友圈原文：用于判断内容语义、场景质量和是否值得关注。
- 抽取结果：用于快速发现 `events`、`signals`、`traits`、`rels`，并辅助筛选历史记录。
- customers 历史：用于召回相关背景；日报场景保持简短，深度场景可展开完整画像、趋势和关系。
- 群聊/私聊：仅在出现明确外部客户、渠道、约定、成交、待办时作为聊天补充。

原文和抽取结果都必须看。抽取结果不得替代原文判断；原文判断也不得忽略抽取出的事件和信号。

标记了 `signals` 的记录必须重点回查原文和 customers 历史。只有原文支持该信号时，才可进入日报或行动建议。

## 分析场景

默认执行 `每日朋友圈雷达`。当用户未指定分析类型时，使用该场景。

支持以下场景框架：

- 每日朋友圈雷达：筛出当天少量值得关注的朋友圈动态，输出轻动作提醒。
- 全量分析：首次数据导入后，对一段时间内的朋友圈、私聊、群聊做客户盘点和关系梳理。
- 群聊分析：分析指定群或近期群聊中的外部客户、渠道、合作、待办和关系变化。
- 深度分析：当用户明确要求综合报告、周报、月报、客户画像或关系网络时使用。

## 业务上下文

围绕以下业务线判断动态价值：

- 定制酒服务：企业活动、婚宴、峰会、宴请、礼品、伴手礼、品鉴、节日礼盒。
- 光明月：高端礼品、品鉴、文化消费、商务接待、重要关系维护。
- 创始合伙人 / 创业投资机会：创业者、项目方、投资人、资源型连接者、城市合伙人、招商加盟、渠道合作。

当动态更像投资、创业、招商、资源链接时，优先从“合伙人 / 项目合作 / 资源连接”角度判断，不要硬转成定制酒机会。

## 每日朋友圈雷达

当用户要求朋友圈日报、今日雷达、微信关系提醒，且未指定深度分析时，执行本场景。

### 数据优先级

1. 朋友圈原文和朋友圈抽取结果。
2. customers 历史记录。
3. 群聊/私聊补充。

群聊/私聊没有有效外部补充时，只在摘要中计数，不展开章节。

### 入选规则

每日最多输出 5 条值得关注动态。

入选内容必须满足至少一项：

- 明确需求：出现采购、预算、宴请、送礼、会议、活动、品鉴、加盟、代理、合伙、投资等可跟进信号。
- 抽取信号：L1 记录中存在 `signals`，且回查原文后确认不是误判。
- 高价值人物动态：重要联系人发布观点、活动、项目、资源连接、政商活动、文旅消费等值得互动的内容。
- 持续场景：customers 历史显示该人近期多次出现同类主题或场景。
- 关系维护价值：适合通过点赞、评论、轻问维持关系，且内容不低质。

优先选择能产生轻动作的动态。没有可执行动作的内容不进入日报。

### 排除规则

排除以下内容：

- 同行酒业广告、品牌宣传、工艺科普、行业月报，除非存在明确渠道合作价值。
- 内部项目进展、研发排障、供应商比选、测试环境、发布流程。
- 普通生活碎片、纯表情、早安晚安、无上下文照片。
- 单纯点赞、评论、关系字段；除非结合原文能形成关系维护动作。
- 仅由关键词命中的弱内容。

同行或内部内容可作为背景，不作为默认日报重点。

### 单条输出规则

每条动态按以下结构输出：

```markdown
**联系人｜主题**
事实：当天发生的具体动态，优先基于朋友圈原文，必要时补充抽取事件。
历史：从 customers 召回一句相关背景；无有效历史则省略。
动作：给出一个轻动作，优先点赞、评论、轻问、观察；仅在明确需求时建议私信。
```

每条最多 3 行。不得写长段分析。不得重复解释业务逻辑。

### 历史召回规则

从 `customers/{wxid}/wechat-analysis/` 读取历史记录。

只召回与当天动态相关的一句话：

- 持续主题：近期多次出现同一类内容。
- 历史事件：过去出现过相似活动、项目、合作、消费场景。
- 互动关系：近期有稳定互动对象，且对当天动作有帮助。
- 历史信号：过去出现过明确需求或约定。

无有效历史时省略 `历史` 行。不得为了完整格式强行编写历史。

### 动作规则

动作只选一个。

动作类型：

- 点赞：弱关系、生活动态、低风险维护。
- 评论：观点、活动、长期关系、高价值动态。
- 轻问：存在场景但需求不明确。
- 私信：明确需求、约定、成交、强机会。
- 观察：有价值但不适合当天触达。

默认使用轻动作。不要急于销售，不要把所有内容导向产品推荐。

### 聊天补充规则

仅当群聊或私聊出现以下内容时，输出聊天补充：

- 外部客户提出需求。
- 外部渠道出现合作入口。
- 已有成交、付款、开票、交付、复购线索。
- 明确约定、承诺、待办需要跟进。

聊天补充格式：

```markdown
## 聊天补充

**对象｜事项**
事实：聊天中出现的外部机会或待办。
动作：下一步跟进动作。
```

没有有效聊天补充时，不输出 `聊天补充` 章节。

### 输出格式

每日朋友圈雷达默认输出：

```markdown
# 今日朋友圈雷达 — YYYY-MM-DD

> 朋友圈 N 条｜聊天补充 N 条｜建议动作 N 条

## 值得关注

**联系人｜主题**
事实：……
历史：……
动作：……
```

禁止输出以下默认章节：

- 概览长文
- 事件分级
- 新特征
- 新关系
- 洞察
- 销售机会
- 今日行动建议
- 今天值得联系的人
- 今天可做
- 不必管

这些章节只在用户明确要求深度分析、周报或月报时使用。

### 质量检查

输出前检查：

- 是否最多 5 条。
- 是否每条都有事实和动作。
- 是否仅在有有效历史时写历史。
- 是否没有重复的人和事章节。
- 是否没有表格。
- 是否没有强行销售。
- 是否没有把内部事项当作市场机会。
- 是否能在 1 分钟内读完。

## 全量分析

当用户要求首次导入分析、全量客户盘点、历史客户梳理时，执行本场景。

执行时读取朋友圈、私聊、群聊原文与抽取结果，并结合 customers 历史记录。输出客户分层、长期场景、明确机会、关系网络和后续动作。本场景可以展开画像、趋势和关系，不使用每日朋友圈雷达的 5 条限制。

---

## 群聊分析

当用户要求分析指定群、跟踪群聊、查找群内机会时，执行本场景。

执行时优先读取群聊原文与群聊抽取结果，再结合相关客户历史。输出群内关键人、外部客户/渠道机会、待办约定、关系变化和需要跟进的具体事项。本场景不使用每日朋友圈雷达的朋友圈优先规则。

---

## 深度分析

当用户要求综合报告、周报、月报、客户画像、关系网络时，执行本场景。

执行时根据用户指定时间范围和数据源展开分析。允许输出事件、画像、关系、洞察、销售机会和行动建议，但必须区分事实、历史和推断。
