# 修改对照格式指南

## 3 种修改格式

根据修改规模选择合适的格式：

| 修改规模 | 推荐格式 | 示例 |
|---------|---------|------|
| <10 行 | 并排对照 | 左右对比，一目了然 |
| 10-50 行 | 精确 diff | 类似 git diff |
| >50 行 | 分次修改 | 每次≤3 处，多次执行 |

---

## 格式 1：并排对照（<10 行）

适用于小规模修改，左右对比清晰。

### 模板

```markdown
## 修改方案

| 位置 | 原内容 | 新内容 |
|------|--------|--------|
| SKILL.md · Step 2 | ```\n检查清单：\n- [ ] 是否复杂\n``` | ```\n检查清单：\n- [ ] 是否违反奥卡姆剃刀\n- [ ] 是否过拟合个例\n- [ ] 是否偏离目标\n``` |
```

### 示例

```markdown
## 修改方案：添加三原则检查

| 位置 | 原内容 | 新内容 |
|------|--------|--------|
| SKILL.md · Step 2 | ```\n## Step 2: 分析问题\n\n分析用户反馈的问题。\n``` | ```\n## Step 2: 原则检查\n\n**目标：** 用三原则分析问题根因\n\n**检查清单：**\n- [ ] 是否违反奥卡姆剃刀？（过度复杂）\n- [ ] 是否过拟合个例？（硬编码数据）\n- [ ] 是否偏离目标？（添加边缘功能）\n``` |
```

---

## 格式 2：精确 diff（10-50 行）

适用于中等规模修改，类似 git diff。

### 模板

```markdown
## 修改方案

**文件：** SKILL.md

**位置：** 第 X-Y 行

```diff
- 删除的内容
+ 新增的内容
  保留的内容
```
```

### 示例

```markdown
## 修改方案：添加迭代原则

**文件：** SKILL.md

**位置：** 第 15-35 行

```diff
  ## 核心原则
  
+ | 原则 | 检查时机 | 说明 |
+ |------|---------|------|
+ | **奥卡姆剃刀** | Step 3/5 | 必要最小化，修改≤3 处 |
+ | **避免过拟合** | Step 2/6 | 泛化为"[企业名称]"仍成立 |
+ | **目标一致性** | Step 2 | 服务于技能初始目标 |
+ 
  ## 迭代流程
  
  ### Step 1: 问题诊断
- 分析用户反馈的问题。
+ **目标：** 理解是个例问题还是共性缺陷
+ 
+ **动作：**
+ 1. 读取用户反馈的具体案例
+ 2. 询问：这个问题出现几次？
+ 3. 判断：个例 or 共性
```
```

## 格式 3：分次修改（>50 行）

适用于大规模修改，拆分为多次执行。

### 模板

```markdown
## 修改方案（分 3 次执行）

### 第 1 次：修改 Step 1-2

**位置：** SKILL.md 第 15-25 行

```diff
...
```

### 第 2 次：修改 Step 3-4

**位置：** SKILL.md 第 26-40 行

```diff
...
```

### 第 3 次：修改 Step 5-6

**位置：** SKILL.md 第 41-60 行

```diff
...
```
```

### 示例

```markdown
## 修改方案：重构整个迭代流程（分 3 次）

### 第 1 次：Step 1-2 问题诊断 + 原则检查

**位置：** SKILL.md 第 15-30 行

```diff
- ## 迭代流程
- 
- 1. 理解问题
- 2. 提出方案
- 3. 执行修改
+ ## 6 步迭代流程
+ 
+ ### Step 1: 问题诊断
+ **目标：** 理解是个例还是共性
+ 
+ ### Step 2: 原则检查
+ **目标：** 用三原则分析根因
```

**等待确认后再执行第 2 次**

---

### 第 2 次：Step 3-4 提出方案 + 备份

**位置：** SKILL.md 第 31-50 行

```diff
+ ### Step 3: 提出修改方案
+ **目标：** 精确方案（≤3 条）
+ 
+ ### Step 4: 备份
+ **目标：** 确保可回滚
```

**等待确认后再执行第 3 次**

---

### 第 3 次：Step 5-6 执行修改 + 测试

**位置：** SKILL.md 第 51-70 行

```diff
+ ### Step 5: 执行修改
+ **目标：** 精确增删改查
+ 
+ ### Step 6: 测试验证
+ **目标：** 验证有效性
```
```

## 格式决策树

```
修改规模？
    ↓
<10 行？
    ├─ 是 → 并排对照格式
    └─ 否 → 继续判断
        ↓
    10-50 行？
        ├─ 是 → 精确 diff 格式
        └─ 否 → 分次修改（每次≤3 处）
```

---

## 修改前确认清单

执行修改前，确认：

- [ ] 修改方案已呈现给用户
- [ ] 用户已确认同意
- [ ] 已执行备份（Step 4）
- [ ] 修改处数≤3 处（奥卡姆剃刀）
- [ ] 修改后技能仍符合三原则

---

## 修改后验证清单

执行修改后，验证：

- [ ] 文件语法正确（无乱码）
- [ ] 中文编码正确（无）
- [ ] 版本号已更新
- [ ] 修改日志已记录
- [ ] 测试案例通过
