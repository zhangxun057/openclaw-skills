# 朋友圈分析报告引擎

从 wechat-moments-preparer 产出的 log 中聚合数据，生成结构化分析报告和行动建议。

## 前置条件

`logs/YYYY-MM-DD.jsonl` 已存在（由 wechat-moments-preparer 产出）。

---

## 数据目录结构

```
moments-analysis/
├── logs/                   ← 分析日志（输入）
│   └── YYYY-MM-DD.jsonl
├── profiles/               ← 用户时间线（输入）
│   ├── wxid_xxx.jsonl
│   └── whitelist.json
└── reports/                ← 分析报告（输出）
    └── report_YYYY-MM-DD.md
```

---

## 执行流程

### Phase 1: 行动建议生成

读取当天 `logs/YYYY-MM-DD.jsonl` 中有 signals 的记录，结合 `profiles/wxid_xxx.jsonl` 历史时间线，生成具体行动建议。

**输入数据：**
- 当天 log 中的 signals（机会线索）
- 对应用户的 profiles 历史时间线（了解背景、关系深度、过往互动）
- 前序行动建议（`reports/actions_YYYY-MM-DD.json`，可能为空）

**行动建议分两类：**

**直接行动（销售导向）** — 有明确 signal 支撑：
- 对象：具体的人（昵称 + wxid）
- 建议：具体动作（发消息/拜访/发方案/邀约品鉴）
- 依据：signal 内容 + 匹配的产品服务（定制酒/创始合伙人）
- 时机：结合 signal 的时间节点（如"下周五峰会前联系"）

直接行动场景示例：

| Signal 类型 | 典型事件 | 建议动作 | 匹配产品 |
|------------|---------|---------|---------|
| 婚礼筹备 | 发布婚纱照/婚宴预告 | 发消息祝贺，附定制婚宴酒方案 | 定制酒（婚宴款） |
| 企业开业/周年 | 发布开业庆典/周年庆 | 发消息祝贺，提供开业/周年定制酒 | 定制酒（企业款） |
| 峰会/论坛 | 发布参会预告/会议现场 | 峰会前联系，提供会议用酒/伴手礼方案 | 定制酒（会议款） |
| 高端消费场景 | 晒豪车/高档餐厅/奢侈品 | 邀约品鉴，推介创始合伙人机会 | 创始合伙人 |
| 商务宴请 | 发布宴请现场/酒局 | 跟进询问用酒需求，推荐定制方案 | 定制酒 |
| 投资/创业 | 发布融资/新项目消息 | 发消息祝贺，介绍创始合伙人机会 | 创始合伙人 |
| 节日送礼 | 临近节日发布礼品相关 | 提前联系，推荐节日定制礼盒 | 定制酒（礼盒款） |

**间接行动（关系维护）** — 无强 signal 但有互动价值：
- 对象：白名单用户或高互动用户
- 建议：评论/点赞/话题连接/转发
- 依据：当天事件 + 历史时间线中的共同话题

间接行动场景示例：

| 用户类型 | 当天内容 | 建议动作 |
|---------|---------|---------|
| 白名单用户 | 发布行业观点/时事评论 | 评论引发话题，建立专业形象 |
| 白名单用户 | 发布生活动态（旅行/美食） | 点赞 + 简短评论，保持存在感 |
| 高互动用户 | 发布与业务相关内容 | 转发或评论，扩大共同话题 |
| 潜在客户 | 发布聚会/活动预告 | 评论互动，为后续直接联系铺垫 |

**排除原则：**
- 同行（白酒/酒业从业者、酒类媒体/咨询）不作为营销对象，可保持普通社交互动但不推进销售

**可用动作类型：**

| 动作 | 说明 |
|------|------|
| 朋友圈评论 | 维持存在感，引发话题 |
| 微信私信 | 有明确 signal 时直接推进 |
| 问候关怀 | 重要关系节点性维护，给情绪价值 |
| 预约拜访 | 高价值客户，有见面机会时推进 |
| 邀约品鉴 | 有消费能力线索，邀请体验产品 |
| 组织聚会 | rels 显示多人有共同关系，可撮合饭局 |
| 设计服务方案 | 有明确需求 signal 后，引导进入方案设计流程 |
| 推荐光明月 | 适合有高端消费/礼品需求的用户 |
| 推荐定制酒 | 适合有企业/婚宴/峰会/宴请场景的用户 |
| 推荐合伙人 | 适合有投资/创业/高净值背景的用户 |

**行动建议原则：**
- 直接行动必须有 signals 支撑；间接行动只需当天有发帖即可触发
- 直接行动优先级高于间接行动
- 同一用户本周已有行动建议的，评估是否需要跟进而非重复建议
- 每条建议必须包含两要素：**动作**（从上表选择）+ **详情**（说什么/怎么做）
- 不生成方案内容本身，只引导用户执行动作；方案生成由后续 skill 负责
- 输出到 `reports/actions_YYYY-MM-DD.json`

### Phase 2: 生成当日报告

基于当天 `logs/YYYY-MM-DD.jsonl` + Phase 1 行动建议，输出 `reports/report_YYYY-MM-DD.md`。

**各章节数据来源：**
- 概览：统计 log 总条数、涉及用户数、有效记录数、signal 数
- 事件识别：从 log 的 `events` 字段聚合，按以下优先级筛选展示：
  - 高价值（重点展示）：政商峰会、政协/工商联活动、企业周年庆典、重大合同签署、高规格仪式、投融资活动
  - 中价值（简要列出）：出行、聚会、生活动态
  - 排除不展示：同行酒业从业者的活动、装修/电梯/家居等普通销售签单
- 新特征捕捉：从 log 的 `traits` 字段提取，**对比 profiles 历史时间线**，只报告首次出现或有变化的特征，重复捕捉的跳过
- 新关系识别：从 log 的 `rels` 字段提取互动对，**对比双方 profiles 历史**，只报告新出现的关系，已知关系跳过
- 行动建议：最多5条；优先顺序：① 有 signal → ② 高价值事件对象 → ③ 中价值事件中的白名单/高互动用户；同行及低端销售人员不出现

## 报告结构

```markdown
# 朋友圈分析报告 — YYYY-MM-DD

> 分析时段：YYYY-MM-DD | N条帖子 | N位用户 | N条有效记录 | N个销售信号

## 概览
本次分析了 YYYY-MM-DD 的 N 条朋友圈，涉及张三、李四等 N 位联系人，其中 N 条包含有价值信息，捕捉到 N 个销售信号。

## 事件
（列出当天发生的值得关注的事件，每人一行）
- 张三：参加博鳌峰会，主题酱香品牌出海
- 李四：签署别墅电梯订单，客户为高端楼盘

## 新特征
（仅列出对比 profiles 历史时间线后首次出现或有变化的特征，重复的跳过）
- 王五：首次出现投资圈社交场景，此前记录均为餐饮类

## 新关系
（仅列出对比双方 profiles 历史后新出现的互动关系）
- 赵六 ↔ 钱七：今日互相评论，此前无互动记录

## 销售机会

（有 signal 支撑的对象，重点展开）

**张三（wxid_xxx）**
机会：参加峰会，内容涉及酱香品牌出海，有定制礼盒需求
匹配产品：定制酒（峰会款）

## 今日行动建议

### 优先：有明确机会的对象

**张三（wxid_xxx）**
动作：微信私信
详情：「张总，看到你在博鳌！我们有峰会场景的酱香定制礼盒，发个方案给你看看？」

动作：设计服务方案
详情：对方回复后跟进，进入定制酒方案设计流程

### 其次：值得关注但暂无直接机会的对象

**李四（wxid_yyy）**
动作：朋友圈评论
详情：「恭喜签单！阅山湖的客户品味都很高」
```

## 业务上下文

- 定制酒服务
- 创始合伙人/创业投资机会

行动建议将用户画像特征、事件与以上产品服务匹配。

---

## 分析原则

1. 概览、画像、事件、关系直接从 log 聚合，不额外发挥
2. 行动建议必须有 log 中的 signals 作为依据，不编造
3. signals 是线索，行动建议是具体动作
