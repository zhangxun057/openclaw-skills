# customer-mgr

客户管理技能（CRM 式），实现客户档案的创建、查询、更新和关联管理。

---

## 核心能力

### 1. 客户创建
**触发：** 用户明确要求创建新客户，或调研/拜访后询问创建

**操作：**
1. 检查客户是否已存在（模糊匹配客户名称）
2. 如果不存在 → 创建客户档案
3. 推荐下一步操作：
   - "是否立即调研？" → 调用 customer-research
   - "是否有拜访记录？" → 调用 sales-visit-summary
   - "先建档，稍后补充"

**示例：**
```
用户：创建一个新客户，贵州 XX 酒业
→ customer-mgr: 检查是否存在 → 不存在 → 创建档案 → 推荐调研
```

---

### 2. 客户查询
**触发：** 用户要求查询客户信息

**操作：**
1. 查找客户档案（按名称模糊匹配）
2. 返回完整信息：
   - 基本信息（6 维度画像）
   - 沟通记录列表
   - 调研报告列表
   - 待办事项

**示例：**
```
用户：查一下贵州 XX 酒业的档案
→ customer-mgr: 查找档案 → 返回完整信息
```

---

### 3. 客户更新（自动触发）
**触发：** sales-visit-summary 或 customer-research 执行后

**操作：**
1. 检查客户是否存在
2. 如果存在 → 更新画像（6 维度）
3. 如果不存在 → 询问是否创建

---

### 4. 客户关联
**触发：** customer-research 或 sales-visit-summary 执行时

**操作：**
1. 检查是否针对已有客户
2. 如果是 → 将结果关联到客户档案
3. 如果不是 → 询问是否创建客户

## 使用示例

### 示例 1：首次创建客户

**用户输入：**
```
创建一个新客户，贵州 XX 酒业
```

**customer-mgr 执行：**
```markdown
1. 检查客户是否存在 → 不存在
2. 创建客户档案目录 → `customers/gui-zhou-xx-jiu-ye/贵州 XX 酒业/`
3. 创建客户画像文件 → `01-客户画像.md`
4. 创建沟通记录目录 → `02-沟通记录/`
5. 创建调研报告目录 → `03-调研报告/`
6. 创建待办事项文件 → `04-待办事项.md`
7. 更新客户索引 → `customer-index.md`
8. 推荐下一步操作：
   - "是否立即调研？" → 调用 customer-research
   - "是否有拜访记录？" → 调用 sales-visit-summary
```

**输出：**
```markdown

**客户名称：** 贵州 XX 酒业
**档案路径：** `customers/gui-zhou-xx-jiu-ye/贵州 XX 酒业/`

**已创建文件：**
- ✅ 01-客户画像.md
- ✅ 02-沟通记录/
- ✅ 03-调研报告/
- ✅ 04-待办事项.md

**推荐下一步：**
1. 🔍 是否立即调研？ → 调用 customer-research
2. 📋 是否有拜访记录？ → 调用 sales-visit-summary
3. ⏸️ 先建档，稍后补充
```

## 拼音转换规则

| 客户名称 | 拼音目录 |
|---------|---------|
| 贵州 XX 酒业 | `gui-zhou-xx-jiu-ye/` |
| 多彩贵州城潮黔门 | `duo-cai-gui-zhou-cheng/` |
| 重庆 XX 集团 | `chongqing-xx-ji-tuan/` |

**转换规则：**
1. 汉字转拼音（无声调）
2. 拼音之间用 `-` 连接
3. 大写英文保持不变

## 与其他技能的联动

### customer-research 联动
```
customer-research 执行调研
    ↓
调用 customer-mgr
    ↓
1. 检查客户是否存在
2. 如果存在 → 关联调研报告 + 更新画像
3. 如果不存在 → 询问是否创建
```

### sales-visit-summary 联动
```
sales-visit-summary 整理拜访摘要
    ↓
调用 customer-mgr
    ↓
1. 检查客户是否存在
2. 如果存在 → 保存沟通记录 + 更新画像
3. 如果不存在 → 询问是否创建
```
