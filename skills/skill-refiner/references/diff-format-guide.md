# 修改格式指南

## 目标

**让 AI 的修改方案清晰、精确、可执行**

---

## 格式要求

### 1. Skill 目录结构（修改后）

**目的：** 让用户了解整体结构变化

**格式：**
```markdown
### 1. Skill 目录结构（修改后）

```
skill-name/
├── SKILL.md
│   ├── ...（原有内容）
│   ├── Step 6: Iterate
│   ├── Step 7: 测试用例验证  ← 新增章节
│   └── Step 8: 版本编号  ← 新增章节
├── scripts/
│   ├── init_skill.py
│   └── package_skill.py
└── references/
```
```

**要点：**
- 使用树状结构
- 标注新增/修改/删除的内容
- 保持简洁（不超过 20 行）

---

### 2. 修改位置定位

**目的：** 精确定位修改位置

**格式：**
```markdown
### 2. 修改位置定位

**文件名：** `SKILL.md`
**位置：** "Step 6: Iterate"章节之后，文件末尾之前
**原文内容：**
```markdown
（原文前 3 行）
...
（原文后 3 行）
```
```

**要点：**
- 指定文件名
- 指定章节位置
- 提供原文上下文（前 3 行 + 后 3 行）

---

### 3. 修改内容（精确表述）

**目的：** 让修改可执行、可验证

**格式：**
```markdown
### 3. 修改内容（精确表述）

**操作：** 插入

**插入位置：** 在"Step 6: Iterate"之后，"Usage Logging"之前

**插入内容：**
```markdown
（完整内容，不超过 50 行）
```
```

**操作类型：**
- **插入** - 新增内容
- **删除** - 删除内容
- **修改** - 替换内容

**要点：**
- 明确指出操作类型
- 精确定位插入位置
- 提供完整的新增内容
- 控制在 50 行以内（奥卡姆剃刀）

---

### 4. 修改理由

**目的：** 让用户理解为什么这样修改

**格式：**
```markdown
### 4. 修改理由

**对应原则：**
- ✅ 就事论事：修改后能解决一类问题（企业个人股东识别）
- ✅ 保持目标一致性：聚焦于"客户信息调研"，不偏离
- ✅ 奥卡姆剃刀：只增加 1 节（13 行），没有更简单的方案
- ✅ 先确认后修改：等待用户确认方案后再执行
```

**要点：**
- 对应 4 大核心原则
- 解释为什么这样修改
- 说明为什么没有更简单的方案

---

## 完整示例

### 示例：为 customer-research 增加风险识别

```markdown
## 修改方案

### 1. Skill 目录结构（修改后）

```
customer-research/
├── SKILL.md
│   ├── ...（原有内容）
│   ├── Step 6: Iterate
│   ├── ## ⚠️ 风险识别原则  ← 新增章节
│   └── Usage Logging
├── scripts/
│   ├── main.py
│   └── report.py
└── references/
```

### 2. 修改位置定位

**文件名：** `SKILL.md`
**位置：** "Step 6: Iterate"章节之后，"Usage Logging"之前
**原文内容：**
```markdown
### Step 6: Iterate

After testing the skill, users may request improvements...

---

## Usage Logging (Required)
```

### 3. 修改内容（精确表述）

**操作：** 插入

**插入位置：** 在"Step 6: Iterate"之后，"Usage Logging"之前

**插入内容：**
```markdown
---

## ⚠️ 风险识别原则

**AI 应主动识别并提示以下高风险信息：**

1. **企业个人股东** - 可能暗示企业资源依赖个人
2. **历史股东频繁变更** - 可能暗示经营不稳定
3. **被执行/失信记录** - 直接影响企业信用
4. **对外投资过多** - 可能暗示主业不聚焦

**AI 应询问：** "这些信息可能影响您的决策，需要我详细说明吗？"
```

### 4. 修改理由

**对应原则：**
- ✅ 就事论事：修改后能解决"不会主动提示风险"的问题（一类问题）
- ✅ 保持目标一致性：聚焦于"客户信息调研"，不偏离
- ✅ 奥卡姆剃刀：只增加 1 节（13 行），没有更简单的方案
- ✅ 先确认后修改：等待用户确认方案后再执行

**测试用例：**
- 企业个人股东识别
- 历史股东频繁变更识别
- 被执行/失信记录识别
```

---

## 禁止的格式

### ❌ 模糊表述

```markdown
优化一下逻辑
改进风险识别部分
让 AI 更智能一些
```

### ❌ 没有定位

```markdown
在适当的位置增加风险识别
修改相关章节
```

### ❌ 超过 50 行

```markdown
（一次性新增 200 行内容）
```

### ❌ 没有理由

```markdown
修改内容：
...

（没有说明为什么这样修改）
```

---

## 格式检查清单

在提供修改方案前，AI 应自检：

- [ ] 提供了 Skill 目录结构（修改后）
- [ ] 精确定位了修改位置（文件 + 章节 + 原文）
- [ ] 修改内容精确到行（增/删/改）
- [ ] 修改内容 ≤ 50 行
- [ ] 说明了修改理由（对应 4 大原则）
- [ ] 提供了测试用例要求
- [ ] 等待用户确认后再执行

---

## 总结

**好的修改方案：**
- ✅ 结构清晰（目录 → 位置 → 内容 → 理由）
- ✅ 定位精确（文件 + 章节 + 原文）
- ✅ 内容简洁（≤ 50 行）
- ✅ 理由充分（对应 4 大原则）
- ✅ 可执行（用户确认后直接修改）

**最终目标：** 让用户完全理解修改内容，拥有决策权
