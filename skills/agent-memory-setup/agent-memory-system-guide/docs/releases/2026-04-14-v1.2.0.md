# v1.2.0 Release Notes

## Summary

This release closes the local-memory workflow loop. The repository now supports session-start recovery initialization, scoped doctor checks, review-ready distillation, and idempotent application into `MEMORY.md`.

## Highlights

- Added `session-start` to initialize the local recovery layer from turn 1.
- Added structured capture metadata including a stable `candidate_document_id`.
- Added `doctor` for active-layer health checks with optional Obsidian vault verification.
- Added `distill` to summarize candidate memory into `suggested_memory`, `recovery_only`, and `follow_up`.
- Added Markdown distill reports with bucketed sections.
- Added `apply` to write distilled long-term memory into `MEMORY.md` with idempotent document-id deduplication.

## Suggested release summary

- Local memory workflow now supports `session-start` -> `distill` -> `apply`.
- Distilled entries use `candidate_document_id` for stable deduplication.
- `doctor` checks only the active local layer unless Obsidian is explicitly enabled.
