# agent-memory-system-guide

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

GitHub repository: [cjke84/agent-memory-system-guide](https://github.com/cjke84/agent-memory-system-guide)

Canonical OpenClaw skill id: `memory-system`

This repository provides a practical local-first Agent memory workflow for OpenClaw, Codex, and Obsidian.
It uses readable files such as `MEMORY.md`, `SESSION-STATE.md`, `working-buffer.md`, `memory-capture.md`, daily notes under `memory/`, and optional durable Obsidian notes via `templates/OBSIDIAN-NOTE.md`.

OpenViking is an optional enhancement for semantic recall and summaries, not a hard dependency and not required for the core workflow.
Treat OpenViking, `memory_search`, or future memory services as optional recall backends behind the local recovery layer.
OpenClaw native memory, diary, grounded recall, and dreaming remain complementary; this repository stays focused on the portable local recovery layer.
