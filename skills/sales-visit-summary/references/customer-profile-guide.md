# 客户画像使用指南

## 6 维度通用框架

所有行业客户画像都遵循 6 个维度：

```
1. 基础信息 — 客户名称、类型、等级、来源、地区
2. 行业偏好与习惯 — 根据行业模板定义（可插拔）
3. 消费与投资特征 — 经济能力/消费风格/决策链
4. 关系与影响力 — 行业身份/社交网络
5. 交互历史 — 沟通记录/交易历史/待办
6. 开放笔记 — 自由记录，不限制
```

---

## 维度 2：行业偏好与习惯（可插拔）

**这是核心差异化维度，根据客户行业选择对应模板：**

| 行业 | 模板文件 | 核心维度 |
|------|---------|---------|
| 酒类运营 | `industry-templates/wine.md` | 香型/品牌/场景/价格/专业程度 |
| 导游行业 | `industry-templates/tourist.md` | 线路/预算/频率/决策特征 |
| 手艺人 | `industry-templates/artisan.md` | 工艺/材质/风格/定制需求 |

---

## 如何选择模板

**步骤：**

1. **识别客户所属行业**
   - 酒类相关 → 酒类运营模板
   - 旅游相关 → 导游行业模板
   - 手工艺相关 → 手艺人模板
   - 其他 → 通用维度（维度 1/3/4/5/6）

2. **选择对应模板**
   - 查看 `references/industry-templates/` 文件夹
   - 选择对应行业的 `.md` 文件

3. **按模板维度收集信息**
   - 在拜访/沟通中针对性提问
   - 将信息存入客户画像维度 2

---

## 信息提取规则

从用户输入中提取：

| 维度 | 提取内容 | 是否必填 |
|------|---------|---------|
| 1. 基础信息 | 客户名称、联系人、联系方式 | ✅ 必填 |
| 2. 行业偏好 | 根据行业模板 | ⚠️ 选填（有就记录） |
| 3. 消费特征 | 预算/决策风格/投资经验 | ⚠️ 选填 |
| 4. 关系影响力 | 行业身份/社交网络 | ⚠️ 选填 |
| 5. 交互历史 | 沟通记录/交易历史 | ✅ 自动记录 |
| 6. 开放笔记 | 自由记录 | ⚠️ 选填 |

---

## 缺失维度提示

当客户画像缺少关键信息时，AI 会提示：

```
⚠️ 客户画像缺失维度提示：
- 行业偏好：未提及 [根据模板提示]，建议下次了解
- 消费特征：未提及预算，建议了解
- 关系影响力：未提及行业身份，建议了解
```

---

## 客户画像文件位置

**存储位置：** `storage/{客户 ID}/{客户名称}/01-客户画像.md`

**文件命名：** `01-客户画像.md`

## 示例：酒类运营客户画像

```markdown
# C001-客户名称

## 1. 基础信息
| 字段 | 值 |
|------|------|
| 客户名称 | XX 公司 |
| 客户类型 | 企业 |
| 客户等级 | A |
| 来源渠道 | 转介绍 |

## 2. 酒偏好与习惯（行业模板）
### 香型偏好
- 首选：酱香
- 接受：浓香

### 品牌偏好
- 常喝品牌：茅台、五粮液

### 饮用场景
- 主要场景：商务接待 + 自饮

## 3. 消费与投资特征
### 经济能力
- 年白酒预算：100-500 万

## 4. 关系与影响力
### 行业身份
- 餐饮协会会长

## 5. 交互历史
（沟通记录索引、交易历史）

## 6. 开放笔记
（自由记录）
```
