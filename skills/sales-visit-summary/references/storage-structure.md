# 存储结构说明

## 目录结构

```
workspace/storage/
├── {客户名称拼音}/
│   └── {客户名称}/
│       ├── 01-客户画像.md              ← 客户画像（6 维度框架）
│       ├── 02-沟通记录/
│       │   ├── {YYYYMMDD}-{类型}-原始记录.md
│       │   └── {YYYYMMDD}-{类型}.md
│       ├── 03-订单/
│       │   └── O{序号}-{订单名称}/
│       │       ├── 订单摘要.md
│       │       └── 相关文件/
│       └── 04-待办事项.md
```

---

## 文件夹命名规则

### 一级目录：客户名称拼音

**规则：**
- 使用汉语拼音
- 连字符分隔
- 小写字母

**示例：**
| 客户名称 | 拼音目录 |
|---------|---------|
| 多彩贵州城潮黔门 | `duo-cai-gui-zhou-cheng/` |
| 重庆 XX 集团 | `chongqing-xx-ji-tuan/` |
| 贵州 XX 酒业 | `gui-zhou-xx-jiu-ye/` |

### 二级目录：客户名称全称

**规则：**
- 使用客户工商注册名称
- 如无全称，使用常用简称

**示例：**
| 客户全称 | 目录名 |
|---------|-------|
| 多彩贵州城潮黔门 | `多彩贵州城潮黔门/` |
| 重庆 XX 金融集团有限公司 | `重庆 XX 集团/` |

## 文件命名规则

### 01-客户画像.md

**固定文件名：** `01-客户画像.md`

**内容：** 客户画像 6 维度框架

---

### 02-沟通记录/

**子目录：** 所有沟通记录存放在此目录下

**文件命名：** `{YYYYMMDD}-{沟通类型}-{可选后缀}.md`

**沟通类型：**
| 类型 | 说明 | 示例 |
|------|------|------|
| 上门拜访 | 到客户公司拜访 | `20260327-上门拜访.md` |
| 电话 | 电话沟通 | `20260325-电话.md` |
| 微信 | 微信文字/语音沟通 | `20260324-微信.md` |
| 飞书会议 | 飞书视频会议 | `20260326-飞书会议.md` |
| 饭局 | 商务用餐 | `20260328-饭局.md` |

**后缀：**
- `-原始记录`：用户输入的原始笔记
- 无后缀：整理后的内部摘要

**示例：**
```
02-沟通记录/
├── 20260327-上门拜访.md          ← 内部摘要
├── 20260327-上门拜访 - 原始记录.md  ← 原始记录
└── 20260325-微信.md               ← 早期微信沟通
```

---

### 03-订单/

**子目录：** 每个订单一个子目录

**命名：** `O{序号}-{订单名称}`

**示例：**
```
03-订单/
├── O001-基酒投资引导方案/
│   ├── 订单摘要.md
│   ├── 方案文档.md
│   └── 相关文件/
└── O002-年度框架合作协议/
    ├── 订单摘要.md
    └── 合同草案.md
```

**固定文件名：** `04-待办事项.md`

## 创建目录的 Power Shell 命令

```powershell
# 创建客户目录
$customerName = "多彩贵州城潮黔门"
$pinyin = "duo-cai-gui-zhou-cheng"
$basePath = "C:\Users\44452\.openclaw\agents\guaiguaixia\workspace\storage\$pinyin\$customerName"

New-Item -ItemType Directory -Path "$basePath\02-沟通记录" -Force

```

## 最佳实践

1. **首次拜访前** → 创建客户目录结构
2. **每次拜访后** → 更新客户画像 + 沟通记录 + 待办事项
3. **订单启动时** → 创建订单子目录
4. **定期清理** → 归档已成交/已流失客户
