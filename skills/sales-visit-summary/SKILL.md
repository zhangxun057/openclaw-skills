# sales-visit-summary

拜访后整理客户沟通记录，生成内部摘要、更新客户画像、待办事项和跟进建议。

---

## 核心流程

### Step 1: 整理内部摘要（核心靶子）

从用户输入中提取关键信息，生成结构化摘要：

```markdown
# YYYY-MM-DD 拜访 {客户名称}

**沟通 ID：** COM-{YYYYMMDD}-{序号}
**日期：** YYYY-MM-DD
**类型：** 上门拜访/电话/微信/飞书会议
**负责人：** 张洵
**生成时间：** YYYY-MM-DD HH:MM

---

## 📋 沟通摘要

### 参会人员
- 我方：张洵
- 对方：{姓名}（{职位}）、{姓名}（{职位}）

### 关键讨论要点
1. {话题} — {讨论内容，达成的决定}
2. {话题} — {摘要}

### 客户优先级
- {他们表达的优先级 1}
- {他们表达的优先级 2}

### 疑虑/反对意见
- {疑虑} — {如何处理/状态}

### 行业竞品信息
- {任何竞品提及，说了什么}

### 行动项
| 负责人 | 行动 | 截止 |
|--------|------|------|
| 张洵 | {行动} | {截止} |

### 下一步
- {约定的下一步，含时间线}

### 订单影响
- {这次拜访如何影响订单 — 阶段变化/风险/加速}
```

**存储位置：** `workspace/storage/{客户名称拼音}/{客户名称}/02-沟通记录/{YYYYMMDD}-{沟通类型}.md`

**示例路径：** `workspace/storage/duo-cai-gui-zhou-cheng/多彩贵州城潮黔门/02-沟通记录/20260327-上门拜访.md`

**文件命名规则：**
- 格式：`{YYYYMMDD}-{沟通类型}.md`
- 沟通类型：`上门拜访` / `电话` / `微信` / `飞书会议` / `饭局`
- 示例：`20260327-上门拜访.md`

**如果目录不存在** → 先创建目录 `workspace/storage/{客户名称拼音}/{客户名称}/02-沟通记录/`

---

### Step 2: 更新客户画像（6 维度框架）

根据拜访内容，更新客户画像的 6 个维度。

#### 2.1 查找客户画像文件

**查找路径：** `workspace/storage/{客户名称拼音}/{客户名称}/01-客户画像.md`

**示例路径：** `workspace/storage/duo-cai-gui-zhou-cheng/多彩贵州城潮黔门/01-客户画像.md`

**拼音转换规则：**
| 客户名称 | 拼音目录 |
|---------|---------|
| 多彩贵州城潮黔门 | `duo-cai-gui-zhou-cheng/` |
| 重庆 XX 集团 | `chongqing-xx-ji-tuan/` |
| 贵州 XX 酒业 | `gui-zhou-xx-jiu-ye/` |

**查找逻辑：**
1. **先查找** — 按路径查找 `01-客户画像.md`
2. **如果存在** → 读取文件，更新变化的维度
3. **如果不存在** → 创建新文件（如需背景调研，调用 `customer-research` 技能）

**目录不存在时：**
```powershell
# 创建客户目录结构
New-Item -ItemType Directory -Path "workspace/storage/{客户名称拼音}/{客户名称}/02-沟通记录" -Force
New-Item -ItemType Directory -Path "workspace/storage/{客户名称拼音}/{客户名称}/03-订单" -Force
```

#### 2.2 客户画像模板（新建时使用）

```markdown
# {客户名称} — 客户画像

**最后更新：** YYYY-MM-DD
**首次接触：** YYYY-MM-DD
**拜访次数：** 1

---

## 0. 客户类型
- 类型：{体系伙伴/市场服务商/创始会员/其他会员/定制酒客户/潜在客户}
- 判断依据：{如"餐饮协会会长，首次接触" → 潜在客户}

## 1. 行业特征
- 行业：{白酒/餐饮/金融/零售/...}
- 规模：{年产值/营收/员工数}
- 行业地位：{协会职务/区域影响力/市场地位}

## 2. 决策链
- 最终决策人：{姓名 + 职位}
- 技术评估人：{姓名 + 职位}
- 价格谈判人：{姓名 + 职位}
- 内部盟友：{姓名 + 职位}

## 3. 需求特征
- 核心痛点：{痛点描述}
- 显性需求：{客户明确说的需求}
- 隐性需求：{客户没说但实际需要的}

## 4. 采购特征
- 预算范围：{金额}
- 价格敏感度：{高/中/低}
- 决策周期：{快/中/慢}
- 采购流程：{直接采购/招标/多方对比}

## 5. 合作历史
- 首次接触：{日期}
- 拜访次数：{次数}
- 当前阶段：{初步接触/需求确认/方案评估/谈判/成交}
- 历史合作：{如有}

## 6. 资源价值
- 直接价值：{订单金额潜力}
- 间接价值：{转介绍能力/行业影响力/案例价值}
```

#### 2.4 更新规则

**每次拜访后必须更新：**
- ✅ 拜访次数：+1
- ✅ 最后更新：当前日期
- ✅ 当前阶段：如有变化则更新
- ✅ 合作历史：添加本次拜访摘要

**根据拜访内容更新：**
- ⚪ 行业特征：如获取新信息则更新
- ⚪ 决策链：如发现新角色则添加
- ⚪ 需求特征：如需求深化则更新
- ⚪ 采购特征：如明确预算则更新
- ⚪ 资源价值：如评估变化则更新

**更新示例：**
```markdown
## 5. 合作历史
- 首次接触：2026-03-25
- 拜访次数：2  ← 本次拜访后 +1
- 当前阶段：需求确认 → 方案评估  ← 阶段变化
- 历史合作：
  - 2026-03-25：首次拜访，了解基酒投资需求
  - 2026-03-27：第二次拜访，确认方案方向  ← 本次添加
```

```markdown
## 💬 跟进文案建议

**适用场景：** 微信/电话/邮件/当面沟通

**核心要点：**
1. 回顾讨论要点
2. 确认承诺事项
3. 明确下一步

**文案：**
```
{称呼}，您好！

今天非常感谢您的时间！

简单回顾一下今天讨论的要点：
1. {要点 1}
2. {要点 2}

接下来我会按约定推进：
- {承诺事项 1}
- {承诺事项 2}

如有任何问题，随时联系我。

祝好，
张洵
```
```

---

#### 3.3 跟进文案说明

**定位：** 跟进文案是待办事项的一部分，用于指导后续沟通，**不限定渠道**（微信/电话/邮件/当面均可）。

**是否必须输出：**
- ✅ 首次拜访后 — 建议输出
- ✅ 重要节点后 — 建议输出（方案发送、技术演示、价格谈判）
- ⚪ 日常跟进 — 可省略

**文案模板：** 详见 `references/follow-up-templates.md`

## 采访模式（自动触发）

### 触发条件

**满足任一即触发采访模式：**
1. 输入字数 < 80 字
2. 缺失关键字段 ≥ 1 个（时间/对象/内容/态度）
3. 对象指向性不足（仅有姓氏+职位，如"王总"、"张总"）
4. 用户明确说"帮我问一下"或"采访模式"

**关键字段：** 时间（何时）、对象（公司+姓名/职位）、内容（谈了什么）、态度（对方反馈）

**对象指向性：** "国开机电的王总" ✅ ｜ "王总" ❌ ｜ "张总" ❌

---

### 追问流程

**轮次限制：最多 2 轮**

**第 1 轮：** 问缺失的关键字段，最多 2 个问题  
**第 2 轮：** 补充 1 个最重要的问题，然后强制结束

**强制结束条件（满足任一）：**
- 已完成 2 轮追问
- 已获取时间+对象+内容+态度四要素
- 用户表示"就这些"、"先这样"

**追问话术：**
```
信息收到。为整理更准确的记录，补充一下：

1. {问题1}
2. {问题2}
```

## 🔗 与 customer-mgr 联动

**拜访摘要完成后自动执行：**

```python
# 1. 提取客户名称
customer_name = extract_customer_name(summary)

# 2. 检查客户是否存在
customer = customer_mgr.find_customer(customer_name)

if customer:
    # 3. 客户存在 → 保存沟通记录 + 更新画像
    customer_mgr.link_communication(customer_name, summary)
    print(f"✅ 沟通记录已关联到客户档案：{customer_name}")
else:
    # 4. 客户不存在 → 询问是否创建
    print(f"⚠️  检测到新客户：{customer_name}")
    print("是否创建客户档案？")
    print("  - 回复'创建'或'是' → 调用 customer-mgr 创建")
    print("  - 回复'稍后'或'否' → 仅保存沟通记录")
    
    if user_confirms:
        customer_mgr.create_customer(customer_name)
        customer_mgr.link_communication(customer_name, summary)
```

**联动时机：**
- 拜访摘要整理完成后
- 客户画像更新前

**关联内容：**
1. 沟通记录 → `02-沟通记录/{date}-{type}.md`
2. 画像更新 → 合作潜力维度（最近沟通、沟通重点、下一步）
3. 待办事项 → 根据拜访内容生成跟进建议
