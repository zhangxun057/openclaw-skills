# 客户拜访访谈追踪 Skill

## 核心功能

自动化客户拜访信息收集与记录：

1. 追踪访谈状态（最多5轮对话）
2. 智能提取用户回复中的关键信息
3. 搜索客户背景信息（上市公司、纳税大户、企业规模等）
4. 自动提交完整记录到飞书多维表

## 访谈流程

### 启动访谈

通过定时任务或手动执行：
```bash
bash scripts/start-interview.sh
```

### 访谈问题顺序

1. **基础信息**：今天拜访了哪个客户？您和客户是什么关系？
2. **联系信息**：联系人姓名、职务、联系方式？
3. **客户分类**：客户类型（体系伙伴/市场服务商/创始会员/其他会员/定制酒客户/潜在客户）
4. **拜访内容**：拜访目的、沟通内容摘要、客户需求
5. **意向评估**：意向程度（高/中/低）、下次跟进时间

### 客户背景搜索

当获得客户名称后，自动搜索：
- 是否上市公司
- 纳税规模
- 社保人数/企业规模
- 行业地位
- 其他消费实力和销售机会相关信息

如搜索结果不确定，在访谈中向用户确认。

### 提交条件

满足以下任一条件时自动提交：
1. 访谈轮次达到 3 轮
2. 必填信息已齐备（客户名称、拜访者、客户类型、沟通内容）
3. 用户明确表示"就这样"/"没了"

## 使用方式

每次回复前检查：

1. 读取 `state.json`
2. 如果 `active=true`，进入访谈模式
3. 从用户消息中提取信息更新 `data`
4. 如获得客户名称且 `客户背景信息` 为空，执行搜索
5. `round++`
6. 判断是否满足提交条件
7. 满足则调用 `feishu_bitable_app_table_record.create` 提交
8. 提交后重置 `state.json`

## 脚本

### start-interview.sh

初始化访谈状态，设置 `active=true`。

### submit-record.sh（可选）
